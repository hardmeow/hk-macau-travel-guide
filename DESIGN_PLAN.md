# 香港澳门旅行指南网站 - 详细设计方案

## 项目概述
构建一个互动式的港澳旅行规划网站，基于 2026 年 2 月 14-15 日的实际行程数据。网站应该是一个**移动优先、信息密集但易于导航**的旅行助手。

---

## 设计哲学与风格

### 核心设计理念
**"现代旅行指南"** - 融合实用性与美学，打造一个既能快速查阅信息，又能激发旅行兴趣的平台。

### 视觉风格
- **配色方案**：
  - 主色：海洋蓝 (`#0066CC`) - 代表香港维港、国际化、信任感
  - 辅助色：珊瑚红 (`#FF6B6B`) - 代表澳门、活力、警示信息
  - 中性色：深灰 (`#2C3E50`) 和浅灰 (`#ECF0F1`)
  - 强调色：金色 (`#FFB800`) - 代表奢华、购物、特色

- **字体系统**：
  - 标题：`Poppins` 或 `Playfair Display`（优雅、现代）
  - 正文：`Inter` 或 `Open Sans`（清晰、易读）
  - 数字/时间：等宽字体（`Roboto Mono`）

- **间距与排版**：
  - 使用 8px 网格系统
  - 充分的留白，避免拥挤感
  - 卡片式布局，每个信息块独立且清晰

### 交互设计
- **微交互**：
  - 按钮悬停：背景色渐变 + 轻微放大（scale 1.05）
  - 卡片悬停：阴影加深，向上浮起
  - 标签页切换：平滑过渡（300ms）
  - 时间轴：点击展开详情，有动画反馈

- **响应式设计**：
  - 移动端（< 640px）：单列布局，大字体，易点击
  - 平板端（640px - 1024px）：两列布局
  - 桌面端（> 1024px）：三列或网格布局

## 页面结构与功能模块

### 1. 顶部导航栏（Header）
- **固定顶部**，背景色：白色或浅灰
- **左侧**：网站 Logo + 标题（"港澳行程 2026"）
- **右侧**：
  - 主题切换按钮（日/夜模式）
  - 信息图标（快速导航菜单）

### 2. 英雄区（Hero Section）
- **背景**：高质量香港维港或澳门夜景图片（或生成的抽象图）
- **内容**：
  - 大标题："2026 年 2 月 港澳之旅"
  - 副标题："14-15 日完整行程规划 | 交通指南 | 美食推荐"
  - 行程日期徽章："2月14日 - 2月15日"
  - CTA 按钮："开始探索" 或 "查看完整行程"

### 3. 快速导航卡片（Quick Links）
- **4 个卡片**，水平排列（移动端堆叠）：
  1. 📍 **地铁指南** - 点击进入支付方式对比
  2. 🚌 **去澳门** - 点击进入交通方案对比
  3. 🍽️ **餐饮推荐** - 点击进入美食地点
  4. 🛍️ **购物指南** - 点击进入购物地点

### 4. 行程时间轴（Timeline）
- **竖向时间轴**，左右交替显示事件
- **每个事件包含**：
  - 时间（大字体、突出显示）
  - 地点（带地铁站标识）
  - 活动类型（图标 + 标签）
  - 详细描述（可展开）
  - 交通信息（耗时、费用）

### 5. 交通方案对比（Comparison Tables）
- **两个对比表**：
  1. **地铁支付方式对比**：八达通 vs AlipayHK vs WeChat HK
  2. **去澳门交通对比**：金巴 vs 渡轮 vs 直通巴士
- **表格设计**：
  - 响应式表格（移动端转换为卡片）
  - 突出显示推荐方案（背景色 + 徽章）
  - 每行可点击，展示详细信息

### 6. 地点详情卡片（Location Cards）
- **两个分类**：餐饮 & 购物
- **每个地点包含**：
  - 店铺名称 + 图片
  - 地址 + 营业时间
  - 地铁站信息（距离、出口号）
  - 推荐菜品/产品
  - 从酒店的交通方式
  - 预计停留时间

### 7. 底部信息区（Footer）
- **快速链接**：关于、联系、反馈
- **行程摘要**：
  - 总耗时、总费用估算
  - 关键时间点提醒
- **打印按钮**：一键打印行程表

## 功能模块详解

### 模块 1：地铁手机操作指南
**组件名**：`MetroPaymentGuide`

**功能**：
- 三个标签页，分别展示三种支付方式
- 每个标签页包含：
  - 支付方式图标
  - 操作步骤（分步骤卡片）
  - 优缺点对比
  - 适用场景
  - "立即下载 APP" 按钮（链接到 App Store/Google Play）

**交互**：
- 点击标签页切换，有平滑过渡
- 步骤卡片可展开查看详细说明
- 点击"优缺点"可展开详细列表

### 模块 2：行程时间轴
**组件名**：`ItineraryTimeline`

**功能**：
- 竖向时间轴，显示 14-15 日的所有事件
- 每个事件是一个可交互的卡片
- 点击卡片展开详情（地点、交通、费用等）
- 支持按日期或按类型筛选

**交互**：
- 悬停时卡片浮起，阴影加深
- 点击展开/收起详情，有动画
- 时间轴上的点可点击快速跳转

### 模块 3：交通方案对比
**组件名**：`TransportComparison`

**功能**：
- 两个对比表（支付方式 & 去澳门交通）
- 移动端自动转换为卡片布局
- 每行可点击，展示详细信息
- 推荐方案突出显示（绿色背景 + "推荐" 徽章）

**交互**：
- 点击表格行，展示详细对比信息
- 点击"查看详情"按钮，打开模态框
- 支持按条件筛选（如按价格排序）

### 模块 4：地点详情卡片
**组件名**：`LocationCard`

**功能**：
- 网格布局（2-3 列，响应式）
- 每个卡片包含：
  - 高质量图片（餐厅/店铺）
  - 基本信息（名称、地址、电话）
  - 营业时间
  - 地铁信息（最近站点、出口、步行时间）
  - 推荐项目
  - 从酒店的交通方式
  - "获取方向" 按钮（集成 Google Maps）

**交互**：
- 卡片悬停时显示"获取方向"按钮
- 点击"获取方向"，打开 Google Maps 或导航应用
- 点击电话号码，直接拨号（移动端）


## 颜色与样式规范

### 主要颜色
- **主色（蓝）**：`#0066CC` - 用于按钮、链接、强调
- **辅助色（红）**：`#FF6B6B` - 用于警示、重要信息
- **强调色（金）**：`#FFB800` - 用于推荐、特色
- **背景色**：`#FFFFFF` / `#F8F9FA`
- **文字色**：`#2C3E50` / `#555555`

### 阴影与深度
- **卡片阴影**：`0 2px 8px rgba(0, 0, 0, 0.1)`
- **悬停阴影**：`0 8px 16px rgba(0, 0, 0, 0.15)`
- **深阴影**：`0 12px 24px rgba(0, 0, 0, 0.2)`

### 圆角
- **小圆角**：`4px` - 按钮、小组件
- **中圆角**：`8px` - 卡片、输入框
- **大圆角**：`12px` - 模态框、大组件

---

## 动画与过渡

### 过渡时间
- **快速**：`150ms` - 按钮悬停、简单状态变化
- **标准**：`300ms` - 标签页切换、卡片展开
- **缓慢**：`500ms` - 页面加载、大型动画

### 缓动函数
- **ease-in-out**：标准过渡
- **ease-out**：进入动画
- **ease-in**：退出动画

### 关键动画
1. **卡片悬停**：`scale(1.02) + shadow increase`
2. **按钮点击**：`scale(0.95) + color change`
3. **时间轴展开**：`height expand + opacity fade-in`
4. **页面加载**：`fade-in + slide-up`

## 无障碍与性能

### 无障碍（A11y）
- 所有交互元素都可通过键盘访问
- 使用语义化 HTML（`<nav>`, `<main>`, `<section>`）
- 为图片添加 `alt` 文本
- 确保颜色对比度 ≥ 4.5:1

### 性能
- 图片优化（WebP 格式、懒加载）
- 代码分割（按路由）
- 最小化 JavaScript 包大小
- 使用 CDN 加速静态资源

## 用户流程

1. **用户访问网站** → 看到英雄区和快速导航
2. **点击快速导航** → 跳转到对应模块（地铁指南/去澳门/餐饮/购物）
3. **查看详细信息** → 展开卡片、查看对比表、获取地点详情
4. **规划行程** → 查看时间轴、确认交通方式、预估时间和费用
5. **打印或分享** → 导出为 PDF 或分享链接
